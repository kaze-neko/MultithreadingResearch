#### 1. ОБЩИЕ СВЕДЕНИЯ

В данном репозитории находится исходный код приложения, написанного на Java. Данное приложение генерирует файл с последовательностью натуральных чисел от 1 до N и выполняет их обработку (применение к каждому из чисел одной из предустановленных математических функций) в последовательном или параллельном режиме на выбор. 

Интерфейс приложения консольный: после завершения работы приложение выводит в консоль время обработки в наносекундах. *Генерация файла, его считывание, а также разбиение всего диапазона на подзадачи не учитывается при подсчёте времени!* 

В репозитории также находиться файл **res.xlsx**, в котором в табличном виде и с помощью графиков представлены результаты обработки данных. 

#### 2. ИНСТРУКЦИЯ ПО ЗАПУСКУ

Шаг 1 : Собрать jar файл с помощью Gradle. Файл **build.gradle.kts** представлен в репозитории. 

Шаг 2 : Установить несколько локальных переменных окружения, необходимых для корректной работы приложения и настройки параметров его работы.<br>Переменные окружения:
+ LAB_2_N : задаёт длину последовательности. Пример использования: export LAB_2_N=1000.
+ LAB_2_OPERATION : задаёт используемую для обработки математическую операцию. К предустановленным операциям относятся умножение, возведение в степень, взятие факториала и нахождение N-ого числа Фибоначчи. Эти операции обозначаются значениями переменной MULT, EXP, FACT и FIB соответственно. Пример использования: export LAB_2_OPERATION=EXP.
+ LAB_2_COEFFICIENT : задаёт коэффициент K, используемый при операциях умножения и возведения в степень. Пример использования: export LAB_2_COEFFICIENT=42.
+ LAB_2_MODE : задаёт режим обработки данных. Для выбора режима последовательной или параллельной обработки нужно использовать значения SEQ или PAR соответственно. Пример использования: export LAB_2_MODE=PAR.
+ LAB_2_M : задаёт число создаваемых потоков при параллельной обработке данных. Не используется при последовательной обработке. Пример использования: export LAB_2_M=10.

Шаг 3 : Запуск jar файла с помощью команды *java -Xmx12288m -jar \<your filename>.jar*. Все приведённые в таблице данные были получены при выделении 12 гб оперативной памяти. Изменение этой величины может привести к изменению времени работы или возникновению ошибки нехватки памяти.
